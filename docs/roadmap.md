# ロードマップ

現在リリースされている機能は、このツールの出発点に過ぎない。
色の世界には、まだ可視化されていない豊かな領域が数多くある。

---

## 近い将来（優先度：高）

### 画像プレビュー

実際の画像に対してプロファイルを適用し、変換前後を比較できるようにする。
「このプロファイルに変換したら、あの画像の色はどう変わるか」を
スライダーで確認できるビフォー・アフター表示を目指している。

あわせて、色域外（Out-of-gamut）になる部分をハイライトする機能も加えたい。
何色が失われるか、視覚的に示すことが目的だ。

### インタラクティブな色情報表示

2D 色度図の上でクリックした点の色値（RGB / Lab / XYZ）をツールチップで表示する。
「この座標にある色はどんな色か」を直接確認できるようにする。

### UI/UX 改善

- ショートカットキーへの対応
- ガイドツアー（初回訪問者向けのチュートリアル）
- ダークモードの精度向上
- 多言語対応（まず英語）

## 中期（優先度：中）

### WCS / Windows Color System 対応

Windows のカラーマネジメント基盤である WCS 形式のプロファイルを読み込めるようにする。
Windows 環境で使われるプロファイルへの対応を広げることが目的だ。

### OpenColorIO（OCIO）連携

映像・VFX 業界で広く使われるカラーマネジメントフレームワーク、
OpenColorIO の設定ファイルを読み込んで色変換を可視化できるようにする。
映像プロフェッショナル向けの機能強化として位置づけている。

### 3D LUT 生成

プロファイル間の色変換を 3D LUT（Look-Up Table）として出力できるようにする。
`.cube` 形式で書き出せれば、Resolve や Premiere など映像編集ソフトで
直接利用できるようになる。

### 高度な比較モード

現在の 2 プロファイル比較を 3 つ以上に拡張する。
差分をヒートマップとして可視化し、どの色域で差が大きいかを直感的に示す。

### CMYK 対応

印刷向けの CMYK プロファイルを読み込んで可視化できるようにする。
RGB と CMYK の色域の違いを 3D で比べられるようにすることが目的だ。

## 長期・構想段階（優先度：低）

### Rust / WASM による高速化

ICC パーサーや色域計算の処理を Rust で書き直し、
WebAssembly としてブラウザ上で動かすことを検討している。

大きなプロファイルや複雑な変換でも、遅延なく処理できるようにすることが目標だ。
現状のパフォーマンスで不足を感じる場面が出てきたら、本格的に進める。

### PWA 対応

Progressive Web App としてインストール可能にする。
オフラインでも使える状態にし、デスクトップアプリに近い体験を提供する。

### プロファイル編集（限定的）

ホワイトポイントやガンマカーブを簡単に調整し、
編集後のプロファイルをファイルとして書き出せるようにする。
フルフィーチャーのプロファイル編集ツールは目指さないが、
「少し変えて試す」程度のことができると有用だ。

### ワークスペース保存

複数のプロファイルと比較設定を「ワークスペース」として保存し、
後で再現できるようにする。ブラウザのローカルストレージを活用する予定だ。

## 技術的な改善（継続的）

### テスト基盤

自動テストが現時点では整備されていない。
計算ロジック（色域面積計算、色変換など）にユニットテストを加え、
信頼性を高めていく。

対象ライブラリ: Vitest（ユニット/統合）、Testing Library（コンポーネント）、Playwright（E2E）。
カバレッジ目標: ビジネスロジック層（`lib/`）80% 以上。

テスト対象:

- `icc-parser.ts`
- `gamut-coverage.ts`
- `color-vision.ts`
- `profile-sharing.ts`
- `data-export.ts`

### TypeScript 厳格化

型の精度を上げ、実行時エラーが起きにくい状態に整える。
`strict: true` の有効化と `any` 型の排除が主な作業。

### アクセシビリティ

キーボードだけで操作できる状態を目指す。
スクリーンリーダーへの対応も段階的に進める。
目標は WCAG 2.1 AA 準拠。

### CI/CD・インフラ

GitHub Actions によるビルド・テスト・デプロイの自動化。
エラートラッキング（Sentry）とアクセス解析（Google Analytics 4）の導入。

---

## データ管理機能（中期）

### プリセット管理

よく使うプロファイルをブラウザに保存し、タグ付けと検索ができるようにする。

### 履歴機能

最近使ったプロファイルを LocalStorage に記録し、再アクセスを簡単にする。

### ワークスペース拡張

現在は単一セッション限りの状態管理だが、複数ワークスペースとして保存・復元できる形に発展させる。

## 方針

このツールは「見て、触れて、わかる」という体験を核に置いている。
機能を追加するときは、その機能が色の世界を理解することに
本当に役立つかどうかを基準にする。

複雑さより、一目でわかる直感性を優先する。
